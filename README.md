# Exam Craft - Question paper generator with answer analyzer and BTL classifier

This is a Streamlit dashboard that brings together three deep learning tools built for exam management. It covers automated answer grading, question classification by Bloom Taxonomy level, and question paper generation.

All three models were trained on Kaggle using GPU (GPU T4). The dashboard is designed to run locally on your machine without needing to retrain anything.

## The Three Modules

### Module 1 - Answer Evaluator

Grades sort student answers by comparing them against a reference answer or passage. It uses a Siamese BiLSTM model trained on the SemEval 2013 short answer grading dataset. Link: [SemEval Dataset](https://www.kaggle.com/datasets/uppulurimadhuri/semeval2013-task-7-sag)

The model encodes the question and reference answer together, and the student answer separately, using a shared BiLSTM. It then computes the cosine similarity between the two encodings and scales that into a mark out of whatever maximum you set.

You can give it input in two ways. Through text fields where you type a question, passage, student answer, and max marks to grade a single answer at a time. Or through a CSV upload where you grade an entire answer sheet in one go.

The output shows the similarity score, marks obtained, percentage, and a grade label.

---

### Module 2 - BTL Classifier

Classifies any question by its Bloom's Taxonomy Level. The six levels are Remember, Understand, Apply, Analyze, Evaluate, and Create. The model behind this is a BiLSTM combined with a Conv1D layer, trained on the EduQG dataset. Link:[EduQG Dataset](https://www.kaggle.com/datasets/nlztrk/eduqg-dataset-llm-science-exam-format-34k)

It uses a hybrid approach. Strong keyword rules run first, so questions containing words like "design" are immediately classified as Create, and questions containing "compare" go to Analyze. If no keyword matches, the trained model classifies the question by its overall context. Each result includes the predicted level and a confidence score.

You can type numbered questions directly into the dashboard, for example Q1. What is photosynthesis, or upload a plain text file with one question per line.

The output is a table with BTL levels and confidence scores, a bar chart showing the distribution across levels, and a CSV download of the results.

---

### Module 3 - QP Generator

Generates a complete exam paper with multiple choice questions, short answer questions, and descriptive questions. It uses a T5-small model fine-tuned on SQuAD v1.1 for question generation. Link: [SQuad Dataset](https://www.kaggle.com/datasets/stanfordu/stanford-question-answering-dataset)

MCQs are pulled directly from the RACE dataset. Distractors for each MCQ are extracted from nearby noun phrases in the same passage using spaCy, so no separate model is needed for this part. Link: [RACE Dataset](https://www.cs.cmu.edu/~glai1/data/race/)

Short answer questions are generated by T5. The model receives a passage and a key concept extracted by spaCy, and generates a question targeting that concept.

Descriptive questions are generated by flan-T5-base. It identifies the most information-dense sentence in a passage and generates an open-ended question asking the student to explain or discuss it.

You can generate from a custom passage by pasting text or uploading a text file. No datasets are needed for this. You can also run in dataset mode, which connects to your local RACE and SQuAD folders and generates questions from those.

The output is a formatted exam paper in three sections. The questions are shown without the underlying concepts or answers that were used to generate them.

## Getting the Model Files from Kaggle

The notebooks were built for Kaggle GPU. You train once there, download the output files, and drop them into the models folder. The dashboard then loads them locally on CPU.

### Answer Evaluator

Open notebooks/answer-evaluator.ipynb on Kaggle. The dataset you need is the SemEval SAG dataset, specifically the file called Training_data.csv. Run all cells. When done, two files will be saved in the Kaggle output: best_siamese_lstm_model.pth and preprocessor.pkl. Download both and place them here:

```
models/best_siamese_lstm_model.pth
models/preprocessor.pkl
```

### BTL Classifier

Open notebooks/btl-classifier.ipynb on Kaggle. The dataset you need is the EduQG dataset, specifically the file called eduqg_llm_formatted.csv. Run all cells. Three files will be saved: btl_model.h5, btl_tokenizer.pkl, and btl_label_encoder.pkl. Download all three and place them here:

```
models/btl_model.h5
models/btl_tokenizer.pkl
models/btl_label_encoder.pkl
```

Note: the BTL module also works without a trained model. It falls back to rule-based classification using keyword matching. In the dashboard, simply uncheck "Use trained model" to use that mode.

### QP Generator

Open notebooks/qp-generator-2.ipynb on Kaggle. The datasets you need are SQuAD v1.1 and RACE. Run all cells. A folder called exam_gen_model will be saved in the output. Download it and place the entire folder here:

```
models/exam_gen_model/
```

The folder should contain these files: config.json, model.safetensors, tokenizer.json, tokenizer_config.json, spiece.model, and special_tokens_map.json.

## Local Setup

Step 1. Install the dependencies:

```bash
pip install -r requirements.txt
python -m spacy download en_core_web_sm
```

Step 2. Place your downloaded model files inside the models folder, following the structure described in the section above.

Step 3. Start the dashboard:

```bash
streamlit run app.py
```

The dashboard will open at http://localhost:8501 in your browser.

## Using the Dashboard

The left sidebar lets you switch between the three modules.

**Answer Evaluator**

Go to the Evaluate via Text Input tab to grade a single answer. Fill in the question, reference answer or passage, student answer, and maximum marks, then click Grade Answer.

Go to the Evaluate via CSV Upload tab to grade a full answer sheet. Upload a CSV with columns for the question, passage, student answer, and max marks. The dashboard auto-detects column names so minor naming differences are handled. After grading, you can download the full results as a CSV.

**BTL Classifier**

Go to the Classify via Text Input tab and type or paste numbered questions in the format Q1. Your question here. Go to the Classify via File Upload tab to upload a plain text file instead.

The toggle for Use trained model switches between the deep learning model and the rule-based fallback. Rule-based mode works fine even if you have not downloaded the model files.

**QP Generator**

Go to the Generate from Custom Passage tab to paste any text and generate questions from it. Set how many MCQs, short answer questions, and descriptive questions you want, and the system will generate the paper. No internet or datasets are needed for this once the T5 model is in place.

Go to the Generate from Datasets tab to use RACE and SQuAD as the source material instead.

## Model Summary

| Module           | Architecture              | Framework               | Model files                    |
|------------------|---------------------------|-------------------------|--------------------------------|
| Answer Evaluator | Siamese BiLSTM with GloVe | PyTorch                 | .pth and .pkl                  |
| BTL Classifier   | BiLSTM with Conv1D        | TensorFlow / Keras      | .h5 and two .pkl files         |
| QP Generator     | Fine-tuned T5-small       | PyTorch via HuggingFace | exam_gen_model folder          |

All three models run on CPU locally. No CUDA required. The T5 model takes a few seconds per question on CPU, which is acceptable for generating a paper.

## Training Details

| Module           | Training data                     | Epochs | Key metric          |
|------------------|-----------------------------------|--------|---------------------|

| BTL Classifier   | EduQG, around 34000 questions     | 10     | Validation accuracy |
